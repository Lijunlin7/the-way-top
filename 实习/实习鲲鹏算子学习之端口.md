# 实习鲲鹏算子学习之端口

## 端口

**端口（Port）** 是计算机网络中的一个重要概念，用于区分同一台计算机或网络设备上不同的网络服务或应用程序。

### **1. 基本概念**

- **IP地址** 用来标识网络中的一台设备（如电脑、服务器）。
- **端口号** 用来标识该设备上的具体服务或应用程序（范围：0 ~ 65535）。

**类比**：

- **IP地址** 就像一栋大楼的地址（如“北京中关村1号”）。
- **端口号** 就像大楼里的房间号（如“808房间”是会议室，“80房间”是前台）。

------

### **2. 端口的作用**

- **让多个服务共享同一个IP地址**
  比如，一台服务器可以同时提供：
  - **Web服务**（HTTP，端口 80）
  - **文件传输**（FTP，端口 21）
  - **电子邮件**（SMTP，端口 25）
    客户端通过 **IP + 端口** 访问不同的服务。
- **提高通信效率**
  数据包通过端口号准确送达目标应用程序（如浏览器访问网站时默认用80端口）。

------

### **3. 端口的分类**

#### **（1）按范围分**

- **0 ~ 1023**：**知名端口（Well-Known Ports）**，分配给系统或常用服务（如HTTP 80、HTTPS 443）。
- **1024 ~ 49151**：**注册端口（Registered Ports）**，供用户程序或第三方服务使用（如MySQL默认3306）。
- **49152 ~ 65535**：**动态/私有端口（Dynamic/Ephemeral Ports）**，临时分配给客户端程序使用。

#### **（2）按协议分**

- **TCP端口**：面向连接，可靠传输（如网页浏览、文件下载）。
- **UDP端口**：无连接，速度快（如视频流、DNS查询）。

------

### **4. 常见端口示例**

| 端口号 | 服务/协议 | 用途         |
| :----- | :-------- | :----------- |
| 20/21  | FTP       | 文件传输     |
| 22     | SSH       | 安全远程登录 |
| 25     | SMTP      | 发送邮件     |
| 53     | DNS       | 域名解析     |
| 80     | HTTP      | 网页访问     |
| 443    | HTTPS     | 加密网页访问 |
| 3306   | MySQL     | 数据库服务   |
| 3389   | RDP       | 远程桌面连接 |

------

### **5. 端口相关操作**

#### **（1）查看端口占用（Windows/Linux）**

- **Windows**：

  cmd

  ```
  netstat -ano | findstr "端口号"
  ```

- **Linux/Mac**：

  bash

  ```
  netstat -tulnp | grep "端口号"
  或
  ss -tulnp | grep "端口号"
  ```

#### **（2）开放/关闭端口**

- 需通过 **防火墙** 或 **路由器** 设置（如Windows防火墙、Linux的`iptables`）。

### **6.端口作为间接层**

如果没有这个间接层, 客户端要与服务端通讯, 就要知道服务端对应进程的 ID, 也即是客户端是依赖于服务端的:   显然, 这种模式对于 web 这种一个服务端对应大量客户端访问的情形是极不适应的, 你都不知道有谁可能会来访问你的网站! 你根本无法告诉它们.

而有了端口这一间接层, 对于 web 的情形, 这种依赖被倒置了, 客户端总是把请求发送到 80(或443) 端口, 这些成为标准的一部分, 并要求服务端反过来去适应, 服务端去监听端口的通讯并处理, 变成了一种反向依赖.

如果一个进程想要提供 web 服务, 它启动之后就要去绑定(binding) web 相关的端口,

并在其上持续的监听(listen), 同时在有请求到来的时候去响应(response). 这样一来, 进程 ID 的问题就消解了:

> 这类似于一个接口回调, 浏览器只需要面向接口索取服务, 而无需知道接口服务的具体提供者, 这些细节被端口层所封装并隐藏起来了.

端口这一间接层的存在解耦(decouple)了客户端与服务端之间的强依赖, 整个体系变得很灵活.

> 可以把端口视作一般编程概念中的接口(interface), 而想 Nginx, apache, tomcat 等等可以认为是这个接口的不同实现(Implementation).

类比：

去市民中心办事，去某个窗口办理对应业务, 这个窗口其实就类似于端口了。

那么你要办港澳台通行证, 你就奔向 80 号窗口就完了. 你不要去问门口保卫处的王大爷, 到底是哪位同志办理这个业务.

### 一些疑问

#### 为什么引入端口, 而不是直接使用进程 ID

1. 作为客户端无法知道服务端对应进程的 ID

2. 服务端对应进程重启后 ID 会改变

3. 一个网站的 web 进程 ID 是这个, 另一个网站的可能又是另一个

### 端口设置

#### **1. 端口分类及约束**

| **端口范围**    | **用途**                                            | **能否随意设置？**           |
| :-------------- | :-------------------------------------------------- | :--------------------------- |
| **0~1023**      | 系统保留端口（如 HTTP:80, HTTPS:443, SSH:22）       | ❌ 需 root 权限，一般禁止占用 |
| **1024~49151**  | 注册端口（如 MySQL:3306, Redis:6379），建议避免冲突 | ⚠️ 可设置，但需检查是否被占用 |
| **49152~65535** | 动态/私有端口（临时使用，如客户端随机端口）         | ✅ 可自由使用                 |

------

#### **2. 可以随意设置端口的情况**

 **(1) 开发测试环境**

- 临时启动服务时可用高端口（如 `8080`、`3000`）。

- **示例**：

  sh

  ```
  python -m http.server 9999  # 临时 HTTP 服务在 9999 端口
  ```

 **(2) 自定义内部服务**

- 企业内网的非标准服务（如内部监控工具用 `8888`）。

- **示例**：

  nginx

  ```
  # Nginx 监听非标准端口
  server {
      listen 8888;
      server_name localhost;
  }
  ```

 **(3) 避免冲突的备用端口**

- 默认端口被占用时，改用其他端口（如 MySQL 改用 `3307`）。

- **示例**：

  sh

  ```
  mysqld --port=3307  # 启动 MySQL 在 3307
  ```

------

#### **3. 不可随意设置端口的情况**

 **(1) 与知名服务冲突**

- **问题**：若占用标准端口（如 `80`），会导致系统服务崩溃。

- **示例**：

  sh

  ```
  # 尝试占用 SSH 默认端口 22（需 root 权限）
  my_app --port=22  # 会导致 SSH 服务无法启动！
  ```

 **(2) 防火墙限制**

- **问题**：企业防火墙可能只允许特定端口（如 `443`）对外通信。

- **示例**：

  sh

  ```
  curl https://example.com:12345  # 若防火墙封锁 12345，请求失败
  ```

 **(3) 客户端硬编码依赖**

- **问题**：某些客户端强制连接固定端口（如浏览器默认访问 `80/443`）。

- **示例**：

  url

  ```
  http://example.com:8080  # 用户需手动输入端口，体验差
  ```

 **(4) 安全风险**

- **问题**：使用低端口（如 `0~1023`）需 root 权限，可能引入提权漏洞。

- **示例**：

  sh

  ```
  sudo malicious_app --port=80  # 恶意程序获取 root 权限
  ```

------

#### **4. 如何安全设置端口？**

1. **检查端口占用**：

   sh

   ```
   netstat -tuln | grep 3306  # 查看 3306 是否被占用
   ss -lntp | grep 8080       # 现代替代方案
   ```

2. **避免知名端口**：

   - 参考 [IANA 端口注册列表](https://www.iana.org/assignments/service-names-port-numbers/service-names-port-numbers.xhtml)。

3. **使用高位动态端口**（`49152~65535`）：

   python

   ```
   # Python 随机选择可用端口
   import socket
   sock = socket.socket()
   sock.bind(('', 0))  # 系统自动分配
   print("Port:", sock.getsockname()[1])
   ```

4. **配置防火墙放行**：

   sh

   ```
   sudo ufw allow 3000/tcp  # 允许 3000 端口
   ```
